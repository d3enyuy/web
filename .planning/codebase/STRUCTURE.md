## Directory Purposes

**`/app`**
- Purpose: Next.js App Router pages and root layout
- Contains: Page files (page.tsx), layout file (layout.tsx), metadata generators (robots.ts, sitemap.ts, manifest.ts)
- Key files: `layout.tsx` (root), `page.tsx` (home page)
- Special: Uses App Router pattern; each file maps to a route

**`/components`**
- Purpose: All React components: section components, UI primitives, providers
- Contains: Section components (hero, about, experience, projects, etc.), UI component wrappers, providers (theme, structured data)
- Key files: Hero, About, Experience, Projects, Navigation, ThemeProvider, StructuredData
- Organized: Top-level section and utility components; `/ui` subdirectory for UI library

**`/components/ui`**
- Purpose: Reusable UI component library powered by Shadcn/UI
- Contains: 50+ UI primitives wrapping Radix UI (Button, Card, Dialog, Form, Input, Select, Tabs, Toast, etc.)
- Key files: button.tsx, card.tsx, badge.tsx, dialog.tsx, dropdown-menu.tsx, form.tsx
- Pattern: Each file exports a single component or component family with TypeScript types

**`/hooks`**
- Purpose: Custom React hooks for shared logic
- Contains: useIsMobile for responsive device detection, useToast for toast notifications
- Key files: `use-mobile.ts` (768px breakpoint detection), `use-toast.ts` (notification hook)

**`/lib`**
- Purpose: Utility functions and shared helpers
- Contains: CSS utility functions, possibly type definitions
- Key files: `utils.ts` (cn function for Tailwind class merging)

**`/styles`**
- Purpose: Global CSS and theme configuration
- Contains: Tailwind CSS imports, CSS custom variables for light/dark themes (oklch color format)
- Key files: `globals.css` (root styles with theme variables)
- Special: Uses Tailwind 4 with @custom-variant and CSS variables

**`/public`**
- Purpose: Static assets served directly by Next.js
- Contains: Images (face.JPG, og-image.png), icons (favicon.ico, icon.svg, apple-touch-icon.png), SEO files (robots.txt, sitemap.xml)
- Accessed: Via `/` prefix in URLs

**`/.planning`**
- Purpose: GSD (Generative Software Development) planning and analysis documentation
- Contains: Codebase analysis files (ARCHITECTURE.md, STRUCTURE.md, etc.)
- Special: Generated by GSD mapping tools

## Key File Locations

**Entry Points:**
- `app/layout.tsx`: Root layout with metadata, analytics, theme provider wrapper
- `app/page.tsx`: Home page - composes all section components into single-page experience

## Naming Conventions

**Files:**
- Components: PascalCase + .tsx extension (Hero.tsx, About.tsx, Navigation.tsx)
- Hooks: camelCase with `use-` prefix + .ts extension (use-mobile.ts, use-toast.ts)
- Utilities: camelCase + .ts extension (utils.ts)
- Config files: kebab-case.mjs or .json (next.config.mjs, tsconfig.json)

**Directories:**
- Page routes: lowercase, no spaces (app/, components/, hooks/, lib/)
- UI components: lowercase (ui/)
- Special: No index files in most directories; each component is its own file

**Component Naming:**
- Export function names: PascalCase matching file name (function Hero() in hero.tsx)
- CSS classes: Tailwind utility classes via className prop
- Class merging: Use `cn()` function from `lib/utils.ts` to merge Tailwind classes without conflicts

**Exports:**
- Named exports: Components export named function (export function Hero() {})
- Re-exports: Optional barrel files can be added but not currently used

## Where to Add New Code

**New Section/Page Component:**
- Implementation: Create `/components/{section-name}.tsx`
- Structure: Use `"use client"` directive, style with Tailwind via className, provide section ID matching nav item
- Import: Add import to `app/page.tsx` and add component to main render
- Navigation: Add nav item to `navigation.tsx` navItems array

**New UI Component:**
- Implementation: Create `/components/ui/{component-name}.tsx`
- Pattern: Wrap Radix UI primitives, use `cn()` for class merging, export both component and helper types
- Re-export: Optionally add to barrel file if building component suites

**New Hook:**
- Implementation: Create `/hooks/use-{hook-name}.ts`
- Pattern: Export single function with camelCase name starting with `use`, return relevant state/methods
- Usage: Import into components needing the logic

**Utility Functions:**
- Shared helpers: Add to `/lib/utils.ts` or create new file in `/lib` if large
- Pattern: Named exports, TypeScript types, pure functions preferred

**Styling Changes:**
- Global styles: Add to `/styles/globals.css`
- Theme variables: Update CSS custom properties in `:root` and `.dark` selectors
- Component styles: Use Tailwind classes inline via className prop

**Configuration Changes:**
- TypeScript paths: Update `tsconfig.json` paths section for new import aliases
- Next.js features: Update `next.config.mjs` (output format, image optimization, etc.)
- Shadcn/UI components: Update `components.json` aliases if adding new @ paths

## Special Directories

**`node_modules/`**
- Purpose: Installed npm packages (Next.js, React, Tailwind, Radix UI, etc.)
- Generated: Yes (created by `npm install` or `bun install`)
- Committed: No (.gitignore excludes)
- Size: Large (229 entries); contains all dependencies

**`/public`**
- Purpose: Static assets served by Next.js with no processing
- Generated: No (manually added or committed)
- Committed: Yes (images, icons, SEO files)
- Access: Via `/` prefix in URLs

**`/.planning/codebase/`**
- Purpose: GSD documentation (architecture, structure, conventions, testing, concerns)
- Generated: Yes (by GSD mapping tools)
- Committed: Yes (documentation for future reference)

**`.next/`**
- Purpose: Next.js build output directory
- Generated: Yes (created by `next build` or `next dev`)
- Committed: No (.gitignore excludes)
- Special: Rebuilt on every dev server start or production build
